# 自己進化チェック

作業完了時にこのコマンドを実行し、改善できることがないか振り返る。

## 実行手順

### 1. 今回の作業を振り返る
- ハマったポイントはあったか？
- 同じ問題が過去にも発生していたか？（docs/lessons-learned.md を確認）
- もっと効率的なやり方はあったか？
- 次回同じ作業をする時に知っておくべきことはあるか？

### 2. 自由に更新（承認不要）

該当があれば即座に更新する:

**docs/lessons-learned.md**
- 新しい教訓を追記（日付 + 内容 + 解決策）

**docs/progress.md**
- 今回の作業内容を追記

**docs/architecture.md**
- 技術構成に変更があった場合に更新

**CLAUDE.md**
- 技術スタックやURL等の事実情報に変更があった場合に更新

### 3. 提案が必要な改善（承認必要）

以下に該当する場合は、完了報告に【進化提案】を追加する:

- `.claude/rules.md` の行動原則を変えたい
- `.claude/commands/*` のコマンドを追加・変更したい
- `.claude/agents/*` のエージェントを改善したい
- `.claude/settings.json` の権限を変えたい

**提案フォーマット:**
```
【進化提案】
■ 対象ファイル: [パス]
■ 変更内容: [具体的に何を変えるか]
■ 理由: [なぜ必要か]
■ 影響範囲: [全リポ or このリポのみ]
```

### 4. evolution-log.md に記録

更新した内容を docs/evolution-log.md に記録する:
```
## {日付}
- **変更**: [何を変えたか]
- **理由**: [なぜ変えたか]
- **承認**: 不要 / トム承認待ち / 承認済み
```

### 5. pushに含める

docs/ の更新は今回の作業ブランチに含めてpushする。
別ブランチにする必要はない。
