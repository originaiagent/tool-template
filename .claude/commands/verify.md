# 自己検証（push前に必ず実行）

このコマンドを実行して、push前の品質チェックを行う。
問題が見つかったら自分で修正してから再度実行する。

## 実行手順

### 1. ビルド検証
プロジェクトの種別に応じて実行:
- Next.js: `npm run build`
- Python: `python -m py_compile {エントリポイント}` + `pip install -r requirements.txt`
- TypeScript: `npx tsc --noEmit`

ビルドエラーがあれば修正する。

### 2. dev server起動と動作確認
- Next.js: `npm run dev` でサーバー起動
- Streamlit: `streamlit run {エントリポイント} --server.headless true`
- Node.js: `npm run start` または `npm run dev`

起動後:
- `curl http://localhost:{ポート}` でトップページが200を返すことを確認
- 今回変更した主要ページに `curl` でアクセスして200を返すことを確認
- レスポンスにエラーメッセージが含まれていないことを確認

### 3. API検証（該当する場合）
- 新規・変更したAPIエンドポイントに `curl` でリクエスト
- レスポンスが期待通りの形式であることを確認
- 500エラーが返らないことを確認

### 4. 回帰チェック
- 変更していない主要ページに `curl` でアクセス
- 200が返ることを確認

### 5. 結果の報告

すべてパスした場合:
```
■ 自己検証結果
- ビルド: ✅ 成功
- 画面表示: ✅ 正常（確認ページ: /xxx, /yyy）
- 変更機能の動作: ✅ 正常
- コンソールエラー: ✅ なし
- 既存機能の回帰: ✅ 問題なし
```

問題があった場合:
- 自分で修正して再度検証を実行
- 3回修正しても直らない場合はエスカレーション
